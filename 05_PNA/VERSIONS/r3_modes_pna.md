# PNA and R3 Modes

Статус: migrated from memory / needs verification

## Назначение

Этот файл отдельно фиксирует старую связку PNA и режимов R3.

В общем PNA full эти режимы уже упомянуты, но здесь они вынесены как отдельный migration node.

---

## 1. Общая идея

R3 не является одним однородным состоянием.

В старых обсуждениях он был разделён на режимы:

- R3.1;
- R3.2;
- R3.3.

Эти режимы описывают разные способы работы с ошибкой, вниманием, прогнозом и ретроспекцией.

---

## 2. R3.1 — online predictive control

R3.1 удерживает текущий predictive loop.

Функция:

- поддерживать текущую модель ситуации;
- сканировать несоответствия;
- замечать mismatch;
- не разрушать действие без необходимости.

Формула:

```text
текущая модель -> ожидание -> сканирование mismatch
```

Связь с PNA:

PNA удерживает рабочий паттерн текущей ситуации достаточно долго, чтобы mismatch стал заметен.

---

## 3. R3.2 — local error explanation

R3.2 включается, когда mismatch уже обнаружен.

Функция:

- быстро объяснить ошибку;
- найти локальное правило, которое дало сбой;
- предложить коррекцию;
- вернуть систему к действию.

Формула:

```text
mismatch -> локальное объяснение -> коррекция действия
```

Связь с PNA:

PNA удерживает ошибку, контекст и ближайшую альтернативу.

---

## 4. R3.3 — offline retrospective rewrite

R3.3 включается, когда внешняя нагрузка снижена или когда ошибка требует более глубокой переработки.

Функция:

- ретроспективно пересобрать правило;
- связать несколько ошибок;
- найти повторяющийся pattern;
- изменить способ генерации поведения.

Формула:

```text
ошибка -> ретроспекция -> pattern -> перепрошивка правила
```

Связь с PNA:

PNA в таком режиме может работать не как окно внешней ситуации, а как окно удержания внутренней модели и памяти.

---

## 5. Связь с сном, воображением и рефлексией

### Сон

Может поддерживать неадресную офлайн-переработку.

### Воображение

Может поддерживать будущие траектории и внутренние прогоны.

### Рефлексия

Может поддерживать адресный анализ ошибки и правила.

---

## 6. Статус

Связка PNA и R3 modes является interpretive hypothesis.

Она не доказана нейрофизиологически.

Нужны источники по:

- working memory;
- attention switching;
- error monitoring;
- mental simulation;
- retrospective reasoning.
