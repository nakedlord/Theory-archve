# Consciousness Flicker — PNA Migration Draft

Статус: migrated from memory / speculative hypothesis / needs source verification

## Назначение

Этот файл переносит старую PNA-идею о сознании как мерцающем окне восприятия реальности.

Файл не утверждает доказанный нейрофизиологический механизм.

Он фиксирует рабочую гипотезу:

```text
сознание = временная сборка паттерна активации, которая обновляется короткими окнами
```

---

## 1. Окно восприятия

В старых обсуждениях PNA связывался с окном восприятия примерно 2–5 секунд.

Это окно понималось как интервал, в котором система удерживает актуальную ситуацию как связное целое.

Статус:

```text
hypothesis / needs empirical verification
```

---

## 2. Flicker

Flicker — условное название для обновления окна сознания.

Идея:

```text
опыт не течёт как абсолютно непрерывная линия
он может собираться в короткие рабочие окна
```

Это не значит, что сознание буквально включается и выключается.

Это означает, что рабочая сборка опыта может иметь временную структуру.

---

## 3. Амплитуда

Амплитуда — условная сила активации паттерна.

Высокая амплитуда может означать:

- сильный фокус;
- высокую мотивационную значимость;
- эмоциональный вес;
- удержание объекта в рабочем окне.

Низкая амплитуда может означать:

- слабое удержание;
- рассеивание;
- быстрый распад паттерна;
- отсутствие устойчивого выбора.

---

## 4. Когерентность

Когерентность — связность элементов внутри паттерна.

Высокая когерентность:

```text
элементы ситуации удерживаются как единая структура
```

Низкая когерентность:

```text
внимание распадается, элементы не образуют устойчивую ситуацию
```

---

## 5. Связь с ΔT

ΔT — функциональное окно выбора.

PNA flicker может быть одним из когнитивных слоёв, который делает ΔT возможным.

Осторожная формула:

```text
PNA flicker -> temporary situation window -> possible support for ΔT
```

Нельзя писать:

```text
PNA flicker = ΔT
```
