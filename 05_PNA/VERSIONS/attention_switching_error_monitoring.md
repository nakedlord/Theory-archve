# Attention Switching and Error Monitoring — PNA / R Theory Migration Draft

Статус: migration draft / needs source verification

## Назначение

Этот файл переносит старую связку PNA, переключения внимания, mismatch detection и R3.

Он не является доказанным нейрокогнитивным каноном.

---

## 1. Главная идея

R3 требует не только удержания ситуации, но и способности заметить несоответствие.

Для этого нужны:

- внимание;
- переключение внимания;
- мониторинг ошибки;
- удержание ожидания;
- сравнение ожидания и результата.

---

## 2. Predictive loop

В режиме R3.1 система удерживает текущий predictive loop.

Формула:

```text
модель -> ожидание -> результат -> mismatch detection
```

Если результат совпадает с ожиданием, действие продолжается.

Если возникает mismatch, система может перейти к R3.2.

---

## 3. Attention switching

Переключение внимания нужно, чтобы:

- заметить важный сигнал;
- вывести ошибку в фокус;
- переключиться с действия на анализ;
- сравнить альтернативы;
- вернуться к действию после коррекции.

---

## 4. Error monitoring

Error monitoring — функциональная способность заметить, что текущая модель или действие не сработали.

В R-терминах это вход в анализ правила.

Формула:

```text
ошибка -> внимание -> удержание mismatch -> объяснение -> коррекция
```

---

## 5. Связь с PNA

PNA может быть окном, в котором одновременно удерживаются:

- ожидание;
- результат;
- mismatch;
- возможное объяснение;
- корректирующее действие.

Если PNA распадается слишком быстро, ошибка может не стать объектом анализа.

---

## 6. Связь с R3 modes

### R3.1

Сканирование mismatch.

### R3.2

Локальное объяснение ошибки.

### R3.3

Глубокая ретроспектива и поиск повторяющегося pattern.

---

## 7. Риски

Нельзя напрямую сводить R3 к attention switching.

Attention switching может поддерживать R3, но R3 включает также правило, смысл, цель и социальные инструкции.
