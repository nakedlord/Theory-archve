# Migration Mode

Дата: 2026-04-25
Статус: active

## Главная задача

Текущий приоритет архива — не развитие новых теоретических веток, а максимально подробный перенос уже существующих наработок в GitHub.

## Правило режима

Пока активен Migration Mode:

```text
не развиваем новые идеи -> переносим существующие материалы -> фиксируем статус покрытия -> только потом продолжаем развитие
```

## Что считается переносом

Переносом считается:

- создание файла по уже обсуждённой теории;
- восстановление структуры старого чата;
- перенос канона из памяти или документа;
- перенос evidence-layer;
- перенос patch history;
- перенос кейсов;
- перенос cross-links;
- фиксация статуса: перенесено / частично / не найдено / требует источника.

## Что не считается переносом

Не считаются переносом:

- создание новой гипотезы;
- развитие новой версии канона без необходимости;
- расширение теории за пределы уже обсуждённого;
- создание новых интерпретаций, если они не нужны для архивирования;
- преждевременный canon update.

## Каноническое правило миграции

Если материал уже был в прошлых диалогах, документах или памяти, он переносится как:

- `CANON.md`, если это уже было принято как канон;
- `VERSIONS/`, если была версия;
- `PATCHES/`, если это было изменение канона;
- `CASES/`, если это пример или проверка;
- `EVIDENCE/`, если это проверка источниками или данными;
- `HYPOTHESES/`, если это не было принято;
- `CROSS_LINKS/`, если это связь между теориями;
- `ARCHIVE/`, если это устаревшая или снятая конструкция.

## Ограничение

Полный архив всех ChatGPT-диалогов напрямую недоступен. Поэтому миграция идёт по слоям:

1. текущая память системы;
2. уже найденные Google Drive документы;
3. текущий GitHub;
4. документы-конспекты;
5. вручную открытые старые чаты, если потребуется.

## Рабочий принцип

Каждый найденный пласт должен получить статус:

```text
not started / partial / transferred / needs source / archived / deprecated
```

## Следующий файл

Основная карта миграции хранится в:

```text
00_META/IMPORT_BACKLOG.md
```
