## Принцип плана

Развитие архива идёт не через накопление заметок, а через последовательность:

```text
гипотеза -> патч -> кейсы -> cross-link -> обновление канона -> новая версия -> audit
```

Каждый шаг должен либо усиливать канон, либо выявлять слабость идеи.

# Current Roadmap

## Шаг 1. Создать checklist candidate -> canon

### Цель

Сделать переход идеи в канон более дисциплинированным.

### Файл

- `00_META/CANDIDATE_TO_CANON_CHECKLIST.md`

### Что должен проверять checklist

- есть ли патч;
- есть ли минимум один кейс;
- есть ли cross-link, если идея связана с другой теорией;
- есть ли ограничение или фальсификация;
- не смешаны ли уровни зрелости;
- не раздувает ли идея канон.

### Критерий завершения

Новая идея не может попасть в канон без короткой проверки.

---

## Шаг 2. Создать PNA audit

### Цель

Разделить поддержанные мосты, свободные интерпретации и неподтверждённые участки.

### Файл

- `05_PNA/AUDIT_PNA_DELTA_T.md`

### Что проверить

- что поддерживают источники по working memory and attention;
- что поддерживает линия temporal integration;
- что даёт predictive processing;
- где начинается наша интерпретация;
- что нельзя считать доказанным.

### Критерий завершения

PNA остаётся гипотезой, но получает честную карту проверки.
