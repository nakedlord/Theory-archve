# Predictions and Non-Predictions — Boundary Saturation

Статус: migration draft / speculative model hygiene

## Назначение

Этот файл переносит и дисциплинирует старую линию Boundary Saturation.

Цель — отделить, что модель может осторожно предсказывать, от того, чего она пока не имеет права утверждать.

---

## 1. Текущий статус модели

Boundary Saturation — speculative interpretive model.

Она не является заменой общей теории относительности.

Она пытается читать горизонтальный режим как режим, где внешне доступное описание определяется границей.

---

## 2. Базовая безопасная формулировка

```text
Boundary Saturation may describe black-hole-like regimes as boundary-dominated accessibility near the compactness threshold.
```

Формула compactness:

```text
chi = 2GE / (c^4 R)
```

Горизонтальный режим:

```text
chi >= 1
```

---

## 3. Возможные prediction candidates

### 3.1 Boundary-dominated description

При приближении к threshold описание системы должно становиться всё более граничным, а не объёмным.

Статус:

```text
interpretive candidate
```

### 3.2 Bottleneck behavior

Если внутренняя актуализация корреляций превышает доступность через границу, должен возникать bottleneck.

Статус:

```text
formal candidate / needs dimension check
```

### 3.3 Page-like behavior

Если информация становится доступной через граничный режим, динамика внешне доступных корреляций может иметь Page-like мотив.

Статус:

```text
candidate / not formalized
```

### 3.4 Redshift-like degradation

Идея деградации выхода при росте compactness может быть связана с redshift-like behavior.

Статус:

```text
speculative analogy
```

---

## 4. Non-predictions

На текущем уровне модель не предсказывает:

- новую физику чёрных дыр;
- замену ОТО;
- точную микрофизику горизонта;
- механизм Хокинговского излучения;
- внутреннее состояние вещества;
- максимальную плотность;
- доказанный механизм испарения;
- наблюдаемые отклонения от известных решений без дополнительной формализации.

---

## 5. Что нужно для настоящих предсказаний

Чтобы prediction candidate стал настоящим предсказанием, нужно:

1. задать размерности `C_b` и `I_dot`;
2. определить, что именно течёт через границу: энергия, информация, корреляции или другая величина;
3. связать boundary load с compactness;
4. проверить совместимость с ОТО;
5. получить наблюдаемо отличимое следствие;
6. указать, что могло бы опровергнуть модель.
