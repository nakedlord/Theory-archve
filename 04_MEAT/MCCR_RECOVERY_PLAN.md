# MCCR Recovery Plan

Статус: active recovery plan
Дата: 2026-04-25

## Назначение

Этот файл задаёт план восстановления MCCR.

MCCR упоминается в старом блоке MEAT / PNA / MCCR, но содержание не восстановлено надёжно.

Главное правило:

```text
do not reconstruct MCCR by invention
```

---

## 1. Current status

Файл-заглушка уже создан:

```text
04_MEAT/VERSIONS/mccr_recovery_draft.md
```

Текущий статус:

```text
recovery draft / needs source
```

MCCR нельзя использовать для выводов.

---

## 2. What is known

Из памяти известно только:

- MCCR был связан с кластером MEAT / PNA / MCCR;
- точная расшифровка неизвестна;
- содержание не восстановлено;
- нельзя определить, это отдельная теория, подмодуль MEAT или связка с PNA.

---

## 3. What must be recovered

Нужно восстановить:

1. расшифровку MCCR;
2. объект теории;
3. основные claims;
4. связь с MEAT;
5. связь с PNA;
6. связь с R Theory;
7. статус: active / archived / recovery / separate module;
8. были ли метрики, операторы или формулы;
9. нужны ли отдельные файлы.

---

## 4. Valid sources

Допустимые источники:

- исходный чат MEAT/PNA/MCCR;
- конспект этого чата;
- файл, созданный по этому чату;
- сообщение пользователя с определением MCCR;
- экспорт старого разговора.

Недопустимо:

- угадывать расшифровку;
- выводить MCCR из названия;
- строить новую теорию под старую аббревиатуру;
- использовать MCCR как активный модуль без источника.

---

## 5. Recovery workflow

Когда источник будет найден:

```text
source -> extract exact definitions -> create recovery notes -> classify status -> decide module placement
```

Дальше возможны варианты:

### Option A — MCCR as MEAT submodule

Если MCCR описывает механизм меметических комплексов.

### Option B — MCCR as PNA/R bridge

Если MCCR связан с когнитивной рекурсией или conscious regulation.

### Option C — separate module

Если MCCR имеет самостоятельный объект.

### Option D — archive

Если MCCR была устаревшей или слабой конструкцией.
