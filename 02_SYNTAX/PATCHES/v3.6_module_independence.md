# SynTax Patch v3.6 — Module Independence

Статус: active core-architecture patch

## Проблема

Ядро SynTax могло стать слишком зависимым от конкретных context modules.

Если без любимой теории система не может анализировать задачу, значит ядро недостаточно автономно.

## Добавленная конструкция

Module Independence.

## Определение

Core Thinking Architecture должна уметь проходить полный runtime-cycle без подключения любого конкретного context module.

## Каноническое правило

No-module path должен оставаться допустимым маршрутом.

Если подходящего context module нет, нельзя натягивать неподходящий.

## Что считается нарушением

- система не может начать анализ без конкретной рамки;
- Goal Layer или Scheduler завязаны на одну теорию;
- Validation не работает без выбранного модуля;
- любимая теория маскирует слабость ядра.

## Зачем это нужно

Патч отделяет силу core-thinking от силы отдельных теорий.

## Риск

Если слишком часто выбирать no-module path, система может недоиспользовать сильные рамки.

## Связи

- Context Arbitration;
- E11 Context Lock-In;
- Minimal Core;
- Audit Protocol.
