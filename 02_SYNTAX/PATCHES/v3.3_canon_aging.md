# SynTax Patch v3.3 — Canon Aging

Статус: active memory-infrastructure patch

## Проблема

Канон может накапливать устаревшие конструкции.

Если старые элементы не получают статуса, они продолжают смешиваться с активными версиями.

## Добавленная конструкция

Canon Aging.

## Статусы элементов канона

- Active Canon;
- Archived Canon;
- Deprecated Canon.

## Active Canon

Действующие конструкции, используемые в текущей версии.

## Archived Canon

Исторически важные конструкции, которые больше не работают в runtime, но нужны для понимания эволюции системы.

## Deprecated Canon

Конструкции, заменённые более сильными механизмами.

## Каноническое правило

Элементы не удаляются напрямую.

Они переводятся в Archived или Deprecated через patch protocol.

## Зачем это нужно

Патч сохраняет историю развития без загрязнения активного канона.

## Риск

Если всё архивировать слишком часто, канон может потерять преемственность. Нужен clear replacement rationale.

## Связи
