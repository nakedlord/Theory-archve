# SynTax Patch v1.9 — Infrastructure Split

Статус: active architecture patch

## Проблема

В ранней архитектуре SynTax канон, рабочие идеи, мутации и механизмы изменения могли смешиваться.

Это создавало риск: полуготовые идеи попадали в память как будто они уже признаны.

## Добавленная конструкция

Infrastructure разделяется на два подслоя:

- Memory Infrastructure;
- Evolution Infrastructure.

## Memory Infrastructure

Отвечает за хранение, связность и доступность уже признанных конструкций.

Включает:

- Canon;
- Field;
- Goal Memory;
- карты узлов, статусов и связей.

## Evolution Infrastructure

Отвечает за мутацию, отбор, патчинг и отклонение конструкций.

Включает:

- Idea Bank Engine;
- Sync Protocol;
- Patch Mechanism;
- процессы отбора и проверки.

## Каноническое правило

Новая идея не попадает в Memory Infrastructure напрямую.

Она сначала проходит через Evolution Infrastructure.

## Зачем это нужно

Патч защищает канон от загрязнения рабочими гипотезами.

## Риск

Если разделение станет слишком жёстким, система может замедлить развитие. Поэтому нужен понятный маршрут: hypothesis -> patch -> canon.
