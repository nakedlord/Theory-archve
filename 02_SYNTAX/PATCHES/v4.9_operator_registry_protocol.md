# SynTax Patch v4.9 — Operator Registry Protocol

Статус: active operator-layer patch

## Проблема

Операторы мышления могли использоваться неявно, дублироваться или пересекаться без контроля.

Это создаёт operator explosion и снижает auditability.

## Добавленная конструкция

Operator Registry Protocol.

## Определение

Operator Registry — формальный реестр всех операторов SynTax с их функциями, условиями применения, рисками и validation hooks.

## Паспорт оператора

Каждый оператор должен иметь:

- Operator ID;
- name;
- function;
- target objects;
- preconditions;
- transformation rule;
- expected output;
- compatible navigation commands;
- validation hooks;
- failure modes;
- overlap risks;
- status.

## Статусы операторов

- Core Operator;
- Auxiliary Operator;
- Experimental Operator;
- Deprecated Operator;
- Archived Operator.

## Каноническое правило

Новый оператор нельзя добавлять, если его distinguishing gain не превышает overlap с уже существующим оператором.

## Зачем это нужно

Патч делает Operator Layer управляемым каталогом, а не набором неявных привычек.

## Риск

Если реестр вести слишком тяжело, он замедлит работу. Поэтому полные паспорта нужны прежде всего для стабильных и повторяемых операторов.
