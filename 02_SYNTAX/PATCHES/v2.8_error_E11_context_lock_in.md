# SynTax Patch v2.8 — Error E11 Context Lock-In

Статус: active error architecture patch

## Проблема

Одна интерпретационная рамка может начать доминировать и подавлять альтернативные чтения.

При этом вывод выглядит связным, но фактически становится зависимым от одного context module.

## Добавленная конструкция

Ошибка E11 Context Lock-In.

## Определение

Context Lock-In — сбой, при котором система преждевременно фиксируется на одной рамке и перестаёт рассматривать конкурирующие модули или no-module path.

## Симптомы

- один модуль включается по умолчанию;
- альтернативные рамки не рассматриваются;
- словарь модуля начинает выглядеть как нейтральная реальность;
- no-module path исчезает;
- конфликты между рамками сглаживаются.

## Коррекция

- forced Competitive Read;
- module deactivation;
- Explicit Separation;
- Higher-Order Reframing;
- возврат к Context Arbitration.

## Зачем это нужно

Патч защищает SynTax от скрытой догматизации.

## Риск

Если E11 применять слишком часто, система может чрезмерно сомневаться даже там, где рамка очевидна.
