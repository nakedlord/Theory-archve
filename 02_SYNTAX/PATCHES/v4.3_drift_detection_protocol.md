# SynTax Patch v4.3 — Drift Detection Protocol

Статус: active audit patch

## Проблема

Система может медленно отклоняться от канона без явной ошибки.

Такой drift опасен: поведение формально похоже на нормальное, но постепенно теряет дисциплину.

## Добавленная конструкция

Drift Detection Protocol.

## Определение

Drift — устойчивое смещение reasoning path, validation profile, error pattern или response behavior относительно активного канона.

## Типы drift

- Trace Drift;
- Metric Drift;
- Error Drift;
- Validation Drift;
- User Isolation Drift;
- Context Discipline Drift.

## Каноническое правило

Drift нельзя фиксировать по одному случаю.

Нужно observation window и baseline comparison.

## Severity levels

- D1 Weak Drift;
- D2 Functional Drift;
- D3 Canon Threat.

## Зачем это нужно

Патч делает SynTax чувствительным к медленной эрозии качества.

## Риск

Если baseline выбран плохо, можно принять нормальную адаптацию за drift.
