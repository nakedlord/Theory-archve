# SynTax Patch v3.7 — Error Recurrence Counter

Статус: active error-architecture patch

## Проблема

Одна ошибка может повторяться в разных диалогах и выглядеть как набор локальных сбоев.

Без счётчика повторяемости система не видит системный pattern.

## Добавленная конструкция

Error Recurrence Counter.

## Определение

ERC — счётчик повторений конкретного класса ошибки или сходного symptom pattern в заданном окне наблюдения.

## Что отслеживается

- Error ID;
- recurrence window;
- recurrence count;
- severity drift;
- last seen;
- trigger threshold;
- escalation status.

## Пороги

- Threshold 1: recurring pattern noticed;
- Threshold 2: correction routing required;
- Threshold 3: patch candidate required;
- Threshold 4: audit-level review required.

## Каноническое правило

Повторяющийся сбой нельзя трактовать как набор независимых локальных ошибок.

## Зачем это нужно

Патч превращает повтор ошибок в источник управляемой эволюции архитектуры.

## Риск

Если окно наблюдения слишком короткое, система может видеть pattern там, где был случайный шум.

## Связи

- Error Architecture;
- Audit Protocol;
- Patch Protocol;
- Drift Detection.
