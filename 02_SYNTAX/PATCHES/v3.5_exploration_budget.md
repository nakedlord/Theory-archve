# SynTax Patch v3.5 — Exploration Budget

Статус: active runtime patch

## Проблема

При высокой неопределённости система может бесконечно открывать новые ветви анализа.

Это ведёт к cognitive drift и задержке финального вывода.

## Добавленная конструкция

Exploration Budget.

## Определение

Exploration Budget — допустимый объём дополнительных ветвей анализа до перехода к сужению пространства решений.

## Что влияет на EB

- неопределённость;
- сложность задачи;
- временная глубина;
- насыщенность входа;
- цель работы.

## Каноническое правило

Когда EB исчерпан, Scheduler должен перейти в NARROW или SYNTH.

## Зачем это нужно

Патч удерживает баланс между исследованием и завершением.

## Риск

Если EB слишком мал, система станет преждевременно закрывать анализ.

Если EB слишком велик, система будет расползаться.

## Связи

- Scheduler;
- State Model;
- Operator Selection;
- Complexity Governance.
