# Goal Module v0.1 — Migration Draft

Статус: migrated from memory / active architecture component

## Назначение

Goal Module — слой направленности SynTax.

Он отвечает не за то, какие идеи существуют, а за то, куда должна двигаться работа.

Формула:

```text
Goal Module = Memory of Direction
```

---

## 1. Отличие от других слоёв

Goal Module не равен теме разговора.

Goal Module не равен Idea Bank.

Goal Module не равен Canon.

Goal Module не равен Scheduler.

Различие:

- Idea Bank хранит, что появилось;
- Canon хранит, что признано;
- Goal Module хранит, что должно быть достигнуто;
- Scheduler выбирает, что активно сейчас;
- тема разговора может быть только поверхностным объектом.

---

## 2. Уровни целей

### G0 — векторные цели

Долгие направления развития системы.

Пример: построить единую архитектуру теорий и их проверки.

### G1 — проектные цели

Цели крупных проектов.

Пример: перенести все текущие наработки в GitHub.

### G2 — сессионные цели

Цели конкретного диалога или рабочей сессии.

Пример: перенести SynTax-патчи и Goal Module.

### G3 — операционные цели

Конкретные действия.

Пример: создать файл `GOAL_MODULE.md`.

## 3. Статусы целей

- Active;

---

## 4. Goal Card

Goal Card — единица хранения цели.

Поля:

- ID;
- название;
- уровень;
- родительская цель;
- смысл / зачем;
- критерий завершения;
- статус;
- текущий прогресс;
- блокеры;
- следующий шаг;
- связанные узлы / проекты / канон;
- связанные диалоги;
- последнее обновление.

---

## 5. Session Link

Session Link связывает конкретный диалог с целью.

Поля:

- Active Goal;
- Session Goal;
- Expected Output;
- Why this matters;
- Result;
- Progress Delta;
- Next Step.

---

## 6. Goal State

Goal State — текущая карта активных целей.

Она показывает:

- что сейчас активно;
- что в фоне;
- что заблокировано;
- что завершено;
- какой следующий шаг наиболее полезен.

---

## 7. Канонические правила

Для значимых рабочих диалогов нужно определять:

- Active Goal;
- Session Goal;
- Expected Output.

В конце значимых диалогов нужно фиксировать:

- что продвинуто;
- что осталось открытым;
- какой следующий шаг.

Цель должна иметь критерий завершения.

Цель должна иметь следующий шаг.

Каждый значимый диалог должен давать Progress Delta.

Низкие цели подчинены высоким.

Цель можно явно убить, если она устарела или поглощена другой.

## 8. Текущее применение к GitHub-архиву

### Active Goal

Максимально детализированно перенести текущие теоретические наработки в GitHub.

### Session Goal

Перенести priority-1 migration backlog:

- R Theory v3.1 full;
- SynTax Companion v1;
- Goal Module;
- Culture instruction network full.

### Expected Output

Новые файлы в GitHub, которые фиксируют уже существующие наработки без развития новых теорий.
