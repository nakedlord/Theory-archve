Главная цель:

```text
отделить факт -> источник -> интерпретацию -> R-reading -> статус уверенности
```

---

## 1. Правило источниковой проверки

Для каждого утверждения нужно фиксировать:

```text
claim -> source -> what source actually says -> R interpretation -> confidence status
```

Нельзя использовать внешний источник как доказательство R-теории напрямую.

Источник может поддерживать только отдельный мост.

---

## 2. Confidence statuses

### supported

Источник прямо поддерживает ключевой factual claim, а альтернативные объяснения слабее.

### plausible

Источник совместим с R-reading, но не доказывает его.

### contested

Есть значимые альтернативные объяснения или спор в литературе.

### speculative

Идея пока в основном теоретическая или восстановлена из памяти без достаточной источниковой опоры.

### needs source

Нужен источник до использования claim в сильном выводе.

## 12. What not to do

Не делать:

- не смешивать source claim и R-reading;
- не повышать статус migration draft без проверки;
- не считать источниковую совместимость доказательством теории;
- не переносить speculative claims в canon.
