# Theory Archive

Исследовательский архив для системы взаимосвязанных теорий.

Главный принцип: один репозиторий хранит всю экосистему, а каждая крупная теория ведётся как отдельный проект со своим каноном, версиями, патчами, гипотезами, кейсами, фальсификациями и источниками.

## Структура

```text
00_META/                         # карта всей системы, правила версионирования, общий глоссарий
01_R_THEORY/                     # R-теория / теория рекурсивного выбора и символического разрыва
02_SYNTAX/                       # SynTax как архитектура мышления и управления теориями
03_CULTURE_AS_INSTRUCTIONS/      # культура как вынесенные наружу инструкции
04_MEAT/                         # memetic entanglement / сцепление мемов
05_PNA/                          # окно сознания, PNA и временная сборка восприятия
06_BOUNDARY_SATURATION/          # чёрная дыра как насыщение границы
90_CROSS_LINKS/                  # связи между теориями
99_ARCHIVE/                      # устаревшие версии и снятые конструкции
```

## Правило работы

- `CANON.md` — текущая признанная версия теории.
- `VERSIONS/` — зафиксированные версии канона.
- `PATCHES/` — изменения канона с причиной, эффектом и датой.
- `HYPOTHESES/` — рабочие идеи, ещё не принятые в канон.
- `CASES/` — применения теории к конкретным материалам.
- `FALSIFICATION/` — что может ослабить или опровергнуть теорию.
- `SOURCES/` — источники, данные, статьи, археология, заметки.
- `SYNC_LOG.md` — краткая история рабочих сессий.

## Базовое правило

Новая идея не попадает в канон напрямую. Сначала она существует как гипотеза, затем проходит проверку через кейсы, фальсификацию, связность с другими теориями и только после этого оформляется как патч канона.
